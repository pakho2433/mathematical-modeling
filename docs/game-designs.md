# 六個小學數學建模遊戲設計

每個遊戲都要讓學生經歷「提出方案、測試、比較、修改、解釋」，而不是只回答固定算式。

---

## 遊戲一：公園野餐預算王

**建議年級：小二**  
**數學內容：香港貨幣、加減法、找續、簡單平均分配**

### 情境

媽媽給玩家指定預算，要求玩家在3D公園內購買足夠若干人享用的食物和用品。

### 例子任務

> 你有50元，要為4人準備野餐。最少要有4份飲品、4份食物和1件共享用品。剩餘金錢越多越好。

### 玩家可以做的事

- 自由走到不同店舖查看價格。
- 選擇商品種類及數量。
- 使用不同面額的香港硬幣及紙幣付款。
- 查看總支出、餘額及每人所得物品。
- 儲存方案A和方案B，再作比較。
- 返回媽媽解釋選擇。

### 數學模型

- 總支出 = 各商品單價 × 數量的總和
- 餘額 = 預算 − 總支出
- 每人份數 = 商品總數 ÷ 人數

### 限制條件

- 總支出不可超過預算。
- 每人必須至少有指定數量的食物及飲品。
- 部分商品可能有「買二送一」或限購條件。

### 評分方式

- 完成基本需要：50分
- 沒有超出預算：20分
- 剩餘金錢：最多15分
- 食物種類均衡：最多10分
- 能解釋方案：5分

### 為何屬於數學建模

同一任務有多個合理方案，學生需要建立預算模型、比較方案及作出決定。

## 遊戲二：校園小食部經理

**建議年級：小三**  
**數學內容：乘法、除法、表格、簡單數據分析**

### 情境

玩家管理學校小食部，需要根據過去數天的銷售數據，決定明天每種食物的入貨數量。

### 例子任務

> 過去五天平均每天售出18個麵包，但星期五通常多售出約三分之一。你會為星期五準備多少個？

### 玩家可以做的事

- 查看每天銷量表及柱形圖。
- 選擇每種商品的入貨數量。
- 設定售價或使用指定售價。
- 按「開始營業」觀看模擬結果。
- 查看缺貨、剩貨、收入、成本和利潤。
- 修改入貨方案並再次測試。

### 數學模型

- 預計需求 = 過去銷量的合理估計
- 成本 = 入貨價 × 入貨數量
- 收入 = 售價 × 售出數量
- 利潤 = 收入 − 成本
- 浪費量 = 入貨數量 − 售出數量

### 可變因素

- 天氣
- 星期幾
- 學校活動
- 商品受歡迎程度
- 突發需求

### 評分方式

不只以最高利潤評分，也要考慮缺貨率和浪費率。

### 為何屬於數學建模

學生需要根據不完整數據作出估計，再用模擬結果檢查及修正模型。

## 遊戲三：3D穩定高塔工程師

**建議年級：小四**  
**數學內容：立體圖形、長度、高度、質量、數據記錄**

### 情境

玩家使用有限數量及不同成本的立體積木，建造符合要求的高塔。

### 例子任務

> 使用不多於12件積木，建造一座至少80厘米高、能站立10秒並承托一個球的塔。

### 玩家可以做的事

- 使用正方體、長方體、三角柱、圓柱及球體。
- 旋轉、移動及堆疊物件。
- 按「開始測試」啟動物理模擬。
- 查看高度、底面面積、重心估計、材料數及穩定時間。
- 儲存三個版本並比較。

### 數學模型

- 總高度 = 各層有效高度的總和
- 材料成本 = 各積木成本的總和
- 穩定指標可綜合：底面大小、重心高度、偏離中心程度

### 限制條件

- 積木數量有限。
- 成本不可超過指定預算。
- 高塔必須達到最低高度。
- 必須通過指定時間或承重測試。

### 遊戲記錄

| 版本 | 高度 | 材料數 | 成本 | 穩定時間 | 是否成功 |
|---|---:|---:|---:|---:|---|
| A |  |  |  |  |  |
| B |  |  |  |  |  |
| C |  |  |  |  |  |

### 為何屬於數學建模

學生需要在高度、材料、成本和穩定性之間作取捨，並以測試數據修改設計。

## 遊戲四：3D公園規劃師

**建議年級：小四至小五**  
**數學內容：周界、面積、比例、預算、空間規劃**

### 情境

玩家要在指定土地上設計一個適合居民使用的公園。

### 例子任務

> 在40米 × 30米的土地上，設計一個公園。公園必須有兒童區、草地、洗手間、小食亭及至少兩條相連通道，總成本不可超過80萬元。

### 玩家可以做的事

- 在方格地圖或3D場景拖放設施。
- 改變設施的長、闊及方向。
- 查看佔地面積、周界、通道長度及成本。
- 以第一身視角步行檢查設計。
- 模擬不同人流，觀察擠塞位置。

### 數學模型

- 土地面積 = 長 × 闊
- 設施面積 = 各區面積總和
- 圍欄長度 = 需要圍起部分的周界
- 總成本 = 設施成本 + 道路成本 + 圍欄成本
- 使用率 = 被使用面積 ÷ 總面積

### 限制條件

- 設施不可重疊或超出邊界。
- 通道要連接入口及主要設施。
- 指定比例土地要保留作綠化。
- 洗手間與食物區需要保持合理距離。

### 比較準則

- 是否符合所有基本要求
- 成本
- 綠化面積
- 通道總長度
- 無障礙程度
- 人流是否暢順

### 為何屬於數學建模

學生要把真實規劃問題轉化為面積、距離、成本及人流等數學條件。

## 遊戲五：農田食物鏈模擬器

**建議年級：小五**  
**數學內容：表格、折線圖、變化率、因果關係、預測**

### 情境

農田中有稻米、蝗蟲、青蛙、蛇和麻鷹。玩家要調整各物種初始數量，令生態系統在指定時間內保持平衡。

### 基本關係

- 稻米為蝗蟲提供食物。
- 蝗蟲是青蛙的食物。
- 青蛙是蛇的食物。
- 蛇是麻鷹的食物。
- 動物排泄物會令土壤肥沃，但數量過多也可能破壞環境。

### 玩家可以做的事

- 調整各物種的初始數量。
- 設定稻米生長速度及季節。
- 加入或移除某一種生物。
- 按「運行30天」觀看數量變化。
- 查看折線圖、最高值、最低值及滅絕警告。
- 儲存方案並比較穩定程度。

### 簡化數學模型

每一回合：

- 新數量 = 原有數量 + 出生／生長 − 被捕食 − 自然死亡
- 稻米增長會受土地肥力及蝗蟲數量影響
- 捕食者增長會受獵物數量影響

### 建模任務

> 設計一組初始數量，令所有物種在60天後仍然存在，而且任何一種數量都不可超過環境上限。

### 評分方式

- 所有物種仍然存在
- 波動幅度較小
- 沒有超出環境上限
- 能解釋改變某一數量後造成的連鎖影響

### 為何屬於數學建模

學生透過規則及數據建立動態模型，作出預測，再用模擬驗證。

## 遊戲六：智能交通燈設計師

**建議年級：小六**  
**數學內容：時間、比率、平均數、圖表、最佳化**

### 情境

玩家管理一個十字路口，要調整不同方向的綠燈時間，減少車輛及行人的等候。

### 例子任務

> 早上東西方向每分鐘有20架車，南北方向每分鐘有8架車。請設計一個總長60秒的燈號週期，使平均等候時間最短，同時確保行人有足夠過路時間。

### 玩家可以做的事

- 設定東西及南北方向的綠燈秒數。
- 設定行人過路時間。
- 選擇不同時段的車流數據。
- 運行3D交通模擬。
- 查看平均等候時間、最長車龍及通過車輛數。
- 比較不同燈號方案。

### 數學模型

- 總燈號週期 = 各階段時間總和
- 平均等候時間 = 所有車輛等候時間總和 ÷ 車輛數
- 通過率 = 通過車輛數 ÷ 到達車輛數

### 限制條件

- 燈號週期固定或有上限。
- 每個方向必須有最低綠燈時間。
- 行人必須有安全過路時間。
- 不可同時讓互相衝突的方向通行。

### 為何屬於數學建模

學生要利用車流數據建立時間分配方案，透過模擬尋找較佳解，而不是只有一個固定答案。

# 建議開發次序

## 第一階段：最快完成

1. **公園野餐預算王**：直接升級現有香港貨幣遊戲。
2. **3D穩定高塔工程師**：直接升級現有立體圖形疊高遊戲。
3. **3D公園規劃師**：可重用3D場景及拖放系統。

## 第二階段：需要數據模擬

4. **農田食物鏈模擬器**
5. **校園小食部經理**
6. **智能交通燈設計師**

# 每個遊戲必須加入的共通功能

- 顯示任務及限制條件。
- 數值改變時即時更新算式或數據。
- 可以儲存至少兩個方案。
- 提供「開始測試／運行模擬」。
- 自動生成方案比較表。
- 不直接宣布唯一最佳答案。
- 要求學生輸入或錄音解釋。
- 提供「修改後再測試」。
- 可匯出學生結果供教師評估。
