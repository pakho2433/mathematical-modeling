# 小學數學建模遊戲中心｜Mathematical Modeling Lab

這個專案收集適合香港小學使用的數學建模遊戲、互動模擬及教學活動。

數學建模不只是「計算正確答案」，而是讓學生經歷：

> 現實問題 → 作出假設 → 建立數學表示 → 測試方案 → 檢查結果 → 修改及解釋

## 怎樣才算數學建模？

一個活動最好包含以下元素：

1. **真實情境**：例如購物、建橋、規劃公園、管理生態系統。
2. **限制條件**：例如預算、材料數量、面積、時間或承重量。
3. **多種可行方案**：學生可以用不同方法完成任務。
4. **數學表示**：使用算式、表格、圖像、比例、幾何或數據。
5. **測試與比較**：學生需要判斷哪個方案較好。
6. **修改模型**：根據結果重新設計，而不是只看一次答案。
7. **解釋決定**：學生要說明為何選擇該方案。

## 遊戲及教學內容

| 年級 | 專案 | 數學重點 | 建模任務 |
|---|---|---|---|
| 小二 | 公園野餐預算王 | 香港貨幣、加減、找續 | 在預算內購買足夠物品，並比較不同方案 |
| 小三 | 校園小食部經理 | 乘法、除法、表格 | 預測需求及決定入貨數量，減少缺貨與浪費 |
| 小四 | 3D 穩定高塔工程師 | 立體圖形、測量、數據 | 使用有限材料建造最高而穩定的塔 |
| 小四至小五 | 公園規劃師 | 周界、面積、比例 | 在指定土地及預算內安排設施 |
| 小五 | 農田食物鏈模擬器 | 圖表、變化、因果關係 | 調整物種數量並維持生態平衡 |
| 小六 | 智能交通燈設計師 | 比率、平均數、數據分析 | 調整燈號時間，減少平均等候時間 |

## 現有專案如何升級成數學建模

### 1. P2 Hong Kong Dollar

現有遊戲已包括自由探索、向媽媽接任務、選擇香港硬幣及紙幣付款。

升級方向：

- 不只購買指定物品，改為「50元內準備四人野餐」。
- 同一任務提供多個合理答案。
- 顯示總支出、餘額及每人平均成本。
- 要求學生比較兩個方案並解釋選擇。

專案：<https://github.com/pakho2433/P2-Hong-Kong-Dollar>

### 2. 3D Shape Stacking

現有遊戲已有物理引擎，可測試立體圖形是否站穩。

升級方向：

- 限制積木數量或總成本。
- 設定最低高度及承重要求。
- 自動記錄高度、材料數、倒塌次數及穩定時間。
- 讓學生根據數據修改設計。

專案：<https://github.com/pakho2433/3D-shape-stacking>

### 3. Solid Builder 3D

現有工具可新增頂點、連接稜及即時計算數量。

升級方向：

- 用最少的稜建造符合指定條件的立體。
- 比較不同結構的頂點、稜、面及材料成本。
- 根據正視圖、側視圖和俯視圖還原模型。
- 設計最大內部空間而材料最少的框架。

專案：<https://github.com/pakho2433/solid-builder-3d>

## 建議技術

- **3D及操作**：Three.js
- **物理測試**：Cannon.js 或 Rapier
- **圖表**：Chart.js
- **介面**：HTML、CSS、JavaScript
- **發布**：GitHub Pages
- **數據保存**：LocalStorage 或匯出 CSV

## 每個遊戲的基本畫面

1. 情境及任務說明
2. 可調整的數據或物件
3. 即時數學數據面板
4. 測試／模擬按鈕
5. 方案比較表
6. 修改及再測試
7. 學生解釋欄及結果匯出

## 專案目標

這個專案不是把傳統工作紙搬到3D畫面，而是讓學生能夠：

- 自由探索不同方案；
- 從失敗中修改模型；
- 使用數據支持判斷；
- 將數學應用於真實問題；
- 清楚解釋自己的設計及決定。
